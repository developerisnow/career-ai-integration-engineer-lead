## Основные темы обсуждения

### 1. Использование Claude Code в многопоточном режиме
- **Алекс**: Работает в 3-4 потоках одновременно - DevOps, PR-ревью разработчиков, код-экспорт
- Использует отдельные папки GitWork для разных задач
- Проблемы с управлением множественными потоками и вылетами терминалов

### 2. TMUX для управления агентами
- **Алекс**: Настроил iTerm2 + TMUX для параллельной работы с 4 AI агентами
- Преимущества: синхронизация панелей, поиск по всем панелям, 50k строк истории
- Рабочий кейс: 4 разных репозитория с независимыми агентами

### 3. Лимиты и подписки
- На подписке $100: переключение с Opus на Sonnet при 20% лимита
- На подписке $200: переключение при 50% лимита  
- 5-часовые сессии с лимитами токенов для каждой модели

### 4. Сбор требований и контекст-инжиниринг
**Алмаз** представил футуристический пайплайн:
1. **Сырые требования** - голосовые заметки, неструктурированные данные
2. **Deep Research** - обогащение контекста через множественные источники
3. **Интервью** - формирование GitHub Issues через диалог
4. **Автоматизация** - агенты работают с готовыми Issues в фоновом режиме
5. **Мониторинг** - хуки и уведомления для контроля процессов

### 5. Хуки и уведомления в Claude Code
- Новая функциональность для отслеживания состояний агента
- События: остановка, изменения, уведомления
- Возможность автоматизации через внешние сервисы

### 6. Рефакторинг с AI
**Максим** выделил два типа:
- **Семантический рефакторинг** - работает отлично, нечеткие изменения по описанию
- **Структурный рефакторинг** - проблемы с перемещением файлов, изменением названий
- Необходимость интеграции с Language Server Protocol (LSP)

### 7. Код-ревью и тестирование
- AI хорошо справляется с формальными требованиями и стайл-гайдами
- Claude 4 больше уважает консистентность проекта, чем best practices
- Смещение фокуса на высокоуровневые инструменты проверки

### 8. Версионность промптов
- **Google**: версионность промптов так же важна, как версионность кода
- **Алмаз**: использует нумерацию версий в Claude.md (36-я версия)
- Необходимость тестирования промптов на деградацию
- **E-VALS** как методология для формализованной проверки LLM-систем

### 9. Консюмерское программирование
- Новый термин для описания программирования обычными людьми
- Демократизация разработки через AI-инструменты
- Женщины показывают более аккуратный подход к AI-программированию

### 10. Будущее профессий
- Появление новых ролей: контекст-инжиниринг, сбор требований
- Переквалификация вместо замещения программистов
- Руководители будут инициировать внедрение AI сверху

## Практические инсайты

### Memory Bank организация
- Структура по дням: `2025.07.02-18.56-title.md`
- Ключевые точки, выполненные TODO, итоги сессий
- Использование эмодзи, Mermaid-диаграмм для лучшего восприятия

### GitHub Issues как единая точка
- Сырые требования → структурированные Issues → трекинг прогресса
- Жесткие шаблоны для консистентности
- Интеграция с MCP серверами

### Дашбординг процессов
- Необходимость визуального контроля множественных агентов
- Интеграция с часами, мобильными устройствами
- DORA-метрики для оценки технической эффективности

## Технические детали

### Claude Code особенности
- Автоматическое переключение моделей по лимитам
- Поддержка хуков для автоматизации
- Проблемы с субагентами и API-ключами при подписке

### Инструменты и интеграции
- SpecStory для истории сессий
- Claudia как GUI для Claude Code
- ccusage для мониторинга использования
- GitHub Actions через MCP серверы

## Заключение
Созвон показал эволюцию от простого использования AI-помощников к сложным автоматизированным пайплайнам разработки. Ключевые тренды: контекст-инжиниринг, дашбординг процессов, версионность промптов и смещение роли разработчика к архитектору требований.
